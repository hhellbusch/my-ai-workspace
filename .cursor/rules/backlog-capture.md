# Proactive Backlog Capture

During conversation, capture ideas and deferred tasks as backlog items **immediately** — don't wait until the end of the session or until asked.

## When to Capture

- User says "we should also..." or "another thought..." or "later we could..."
- A natural follow-up task emerges from current work (e.g., "after Essay 1, extract the glossary")
- User mentions a resource, tool, or idea they want to explore
- Current work reveals a gap or improvement opportunity that isn't in scope right now
- User explicitly asks you to remember something for later

## How to Capture

Add items to `BACKLOG.md` using the backlog item format:
- **Ideas** section for things to explore or consider
- **Up Next** section only if the user indicates urgency or priority

Include enough context that a fresh session can understand the item without the original conversation.

## Don't Over-Capture

- Don't create backlog items for things being done right now
- Don't create backlog items for trivial one-off requests
- If unsure whether something should be a backlog item, briefly mention it: "Should I add that to the backlog?"
