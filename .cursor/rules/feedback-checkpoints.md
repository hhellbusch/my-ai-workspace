# Feedback Checkpoints

AI's default mode is agreement. Most people don't habitually ask for criticism. This rule encodes a counter-practice: at natural stopping points, explicitly invite the user's feedback, questions, thoughts, and concerns.

## When to Check In

- **After completing a multi-step task** — Before moving to the next thing, pause and ask.
- **Before committing** — The pre-commit review catches structural issues; a feedback checkpoint catches directional ones. "Does this match what you had in mind?"
- **At session end** — Before creating a handoff or wrapping up.
- **After presenting a plan or design** — Before executing.
- **After substantive content changes** — Especially essays, case studies, or anything that speaks in the author's voice.

## How to Ask

Keep it simple and genuine. Variations:

- "Any feedback, questions, thoughts, or concerns?"
- "Does this match what you had in mind, or should we adjust?"
- "Anything feel off about this direction?"

Don't bury it in a wall of text. Make it visible — at the end, set apart, easy to respond to.

## What This Is Not

- **Not sparring.** `/spar` is structured adversarial review of specific content — counterarguments, severity ratings, documented output. Feedback checkpoints are conversational and lightweight.
- **Not a gate.** Don't refuse to proceed until the user responds. Ask, then continue if the user's response indicates forward momentum.
- **Not performative.** Don't ask after every trivial action. Reserve it for moments where the user's input genuinely matters — directional decisions, content they'll be associated with, work that's hard to undo.

## The Connection

This is the inverse of the sycophancy problem described in [The Shift](../../docs/ai-engineering/the-shift.md) section 6: "The absence of pushback is not the presence of correctness." If the AI never asks and the user never volunteers, issues accumulate silently. Feedback checkpoints create a structural opening for the user to redirect, challenge, or confirm — without requiring them to interrupt.

Sparring challenges the *content*. Feedback checkpoints challenge the *direction*.
