<prerequisite>
**Do not start Phase 4 until Phase 3 is validated.**

Phase 4 assumes Paude can reliably: create a session, execute a bounded task, commit the result, and
produce a clean diff that a human can meaningfully review. If Phase 3 Task 1 did not produce that,
running two sessions in parallel amplifies the failure mode rather than testing multi-agent behavior.

Before starting: confirm `.planning/paude-integration/findings/phase-3-01-findings.md` exists
and contains a successful harvest with a non-trivial, human-reviewable diff. If it doesn't,
return to Phase 3.
</prerequisite>

<objective>
Run the same bounded task with Claude Code (`--agent claude`) and Gemini CLI (`--agent gemini`)
in parallel Paude sessions, harvest both, and compare outputs. The comparison is behavioral —
not just output quality, but how each agent interpreted the spec, where each deviated, and what
precision level each required to produce something useful.

As a secondary test: run two Claude sessions on two *different* tasks concurrently to assess
whether aggregate throughput from parallel same-model instances justifies the overhead.

Purpose: Answer Phase 4 key questions — do different agents produce meaningfully different
outputs for documentation tasks, is multi-agent comparison practical, does it inform the
"AI prioritization bias" backlog item, and does parallelism at two-session scale justify the cost?

Output:
- Two harvest branches (claude output, gemini output)
- Diff between the two branches
- `.planning/paude-integration/findings/phase-4-01-findings.md`
</objective>
