## The comprehension debt spiral

> "The more I lean into this black box, the less I'm able to comprehend what is produced... I find that when I start running [with AI], I can't stop because my context and my grip on the project has completely slipped. And so the easy button becomes more and more easy to touch."

Once AI-generated code grows past what you can fully review (~150 lines per PR for real comprehension), you're dependent on AI to continue — because you can no longer change the code confidently by hand. The spiral:
- Use AI → code you don't fully understand
- Code you don't understand → can't make targeted changes
- Can't make targeted changes → use AI more
- Repeat until you're at the mercy of whatever the next model can do

---

## Two types of easy buttons

**Type 1 (good):** "I know exactly what I want, I just don't know the syntax or API." Specific, precise question from a position of understanding.
> "I don't know how to spawn a process from JS and listen to stdout. Give me an example." This is one of the greatest things ever.

**Type 2 (bad):** "I don't want to do the CSS. Make it for me." Outsourcing the creative/craft work — the iteration cycles that build judgment and taste.
> "I remember when I'd play a section of a game 9,000 times to get exactly the right feel. You can't do that if someone else is making it."

## The code review impossibility (Prime agrees)

> "You can't review 10,000 lines of code a day. You've never done that. You're lying on Twitter for VC clout. If you 'lightly look at it to catch glaringly dumb things,' okay — but you don't know what insane state spaghetti is happening behind those small change sets."

Honest acknowledgment: AI code generation genuinely does outpace human review capacity. The people claiming otherwise are performing confidence, not describing reality.

---

## The vibe coding / evolution analogy (original author's argument)

The original developer argued: vibe coding is like evolution — no master plan, just run against tests, if it passes add to the DNA stack. If evolution can produce a hand without understanding biology, why can't an LLM produce a to-do app without understanding code?

Prime's response: the feedback loop is broken. In evolution, bad DNA means death (organism removed from gene pool). In vibe coding, bad code means users get hacked — and the same bad patterns keep propagating because there's no "don't produce this again" signal.
