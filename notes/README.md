## Adding New Notes

When adding new notes:
1. Create a topical subdirectory if it doesn't exist
2. Use descriptive filenames with kebab-case
3. Update this README with a link to your note
4. Consider whether the note belongs here or in a specific project directory

## Guidelines

- **Quick references** → Keep them here
- **Project-specific docs** → Move to the relevant project (ansible-examples, argo-examples)
- **Technical references** → Consider if they should be in project documentation
