# ClusterCurator — Education Guide for Bare Metal

**Target Audience**: Platform Engineers not yet using ClusterCurator  
**Context**: Bare metal OpenShift clusters managed by RHACM  
**Last Updated**: March 2026

---

## What is ClusterCurator?

**ClusterCurator** is an RHACM component (part of the `cluster-curator-controller`) that lets you run **automated pre- and post-hooks** around cluster lifecycle events:

- **Provisioning** a new cluster (install)
- **Upgrading** a cluster
- **Scaling** a cluster (adding/removing nodes)
- **Destroying** a cluster (decommission)

Hooks run as **Ansible jobs** (via Ansible Automation Platform or AWX) or as directly invoked scripts. This makes ClusterCurator the glue between cluster infrastructure events and your operational automation.

### Without ClusterCurator (Current State)

```
Install cluster → Cluster becomes available → Engineer manually:
                                               1. Logs into console.redhat.com
                                               2. Finds cluster
                                               3. Sets subscription properties
                                               4. Verifies in web console
```

### With ClusterCurator (Target State)

```
Install cluster → Cluster becomes available → ClusterCurator automatically:
                                               1. Calls Ansible job
                                               2. Ansible runs set-ocm-subscription.sh
                                               3. Subscription is set
                                               4. Notification sent to team
```

## How ClusterCurator Works

### Components

```
RHACM Hub Cluster
├── cluster-curator-controller       # Watches ClusterCurator CRDs
├── ClusterCurator CRD               # Per-cluster lifecycle automation config
└── AnsibleJob CRD                   # Generated by curator to invoke automation
         │
         ▼
Ansible Automation Platform (AAP) / AWX
└── Job Templates                    # Your playbooks that run pre/post hooks
         │
         ▼
Post-Install Automation
└── set-ocm-subscription.sh          # Your OCM subscription script
```

### ClusterCurator CRD

The `ClusterCurator` resource lives in the **same namespace as the cluster** on the hub. It defines what automation to run at each lifecycle stage.

```yaml
apiVersion: cluster.open-cluster-management.io/v1beta1
kind: ClusterCurator
metadata:
  name: prod-bm-01                   # Must match the cluster name
  namespace: prod-bm-01              # Must match the cluster namespace
spec:
  desiredCuration: install           # Trigger: install | upgrade | scale | destroy
  install:
    prehook:
      - name: validate-network       # AAP Job Template name
        extra_vars:
          cluster_name: prod-bm-01
    posthook:
      - name: set-ocm-subscription   # AAP Job Template name
        extra_vars:
          cluster_name: prod-bm-01
          support_level: Premium
          usage: Production
  towerAuthSecret: aap-credentials   # Secret with AAP/AWX credentials
```

### Lifecycle Stages

```
┌──────────────────────────────────────────────────────────┐
│                  Cluster Install Flow                     │
├──────────────────────────────────────────────────────────┤
│  ClusterCurator.spec.install.prehook                     │
│  └─ Ansible jobs run BEFORE installation starts          │
│     (e.g. validate DNS, reserve IP, create CMDB record)  │
│                         │                                │
│                         ▼                                │
│  Cluster Installation (Assisted Installer / Hive)        │
│                         │                                │
│                         ▼                                │
│  ClusterCurator.spec.install.posthook                    │
│  └─ Ansible jobs run AFTER installation completes        │
│     (e.g. set OCM subscription, notify team, setup LDAP) │
└──────────────────────────────────────────────────────────┘
```

## Bare Metal Use Case: OCM Subscription via ClusterCurator

### What You Need

1. **Ansible Automation Platform (AAP)** or **AWX** connected to RHACM
2. **Job Template** in AAP with a playbook that calls the OCM subscription script
3. **ClusterCurator** resource deployed alongside each cluster

### Step 1 — Store OCM Token as Kubernetes Secret on Hub

```bash
oc create secret generic rh-ocm-token \
  --from-file=token=/path/to/offline-token.txt \
  -n open-cluster-management
```

### Step 2 — Create AAP Credentials Secret on Hub

```bash
oc create secret generic aap-credentials \
  --from-literal=host=https://aap.example.com \
  --from-literal=token=<aap-api-token> \
  -n prod-bm-01
```

The secret structure ClusterCurator expects:

```yaml
apiVersion: v1
kind: Secret
metadata:
  name: aap-credentials
  namespace: prod-bm-01
type: Opaque
stringData:
  host: "https://aap.example.com"
  token: "your-aap-api-token"
```

### Step 3 — Create the Ansible Playbook (in AAP)

This playbook runs on the RHACM hub (or a jump host) and calls the subscription script:

```yaml
# playbook: set-ocm-subscription.yml
---
- name: Set OCM subscription for new cluster
  hosts: localhost
  gather_facts: false
  vars:
    cluster_name: "{{ cluster_name }}"
    support_level: "{{ support_level | default('Premium') }}"
    service_level: "{{ service_level | default('L1-L3') }}"
    usage: "{{ usage | default('Production') }}"
    system_units: "{{ system_units | default('Cores/vCPU') }}"

  tasks:
    - name: Retrieve OCM token from Kubernetes secret
      kubernetes.core.k8s_info:
        api_version: v1
        kind: Secret
        name: rh-ocm-token
        namespace: open-cluster-management
      register: ocm_secret

    - name: Set OCM token fact
      ansible.builtin.set_fact:
        ocm_token: "{{ ocm_secret.resources[0].data.token | b64decode }}"

    - name: Login to OCM
      ansible.builtin.command:
        cmd: "ocm login --token={{ ocm_token }}"
      no_log: true

    - name: Get cluster ID from OCM
      ansible.builtin.shell: |
        ocm get /api/clusters_mgmt/v1/clusters \
          --parameter "search=name='{{ cluster_name }}'" | \
          python3 -c "import sys,json; print(json.load(sys.stdin)['items'][0]['id'])"
      register: cluster_id_result

    - name: Get subscription ID
      ansible.builtin.shell: |
        ocm get /api/accounts_mgmt/v1/subscriptions \
          --parameter "search=cluster_id='{{ cluster_id_result.stdout }}'" | \
          python3 -c "import sys,json; print(json.load(sys.stdin)['items'][0]['id'])"
      register: sub_id_result

    - name: Patch subscription settings
      ansible.builtin.shell: |
        ocm patch /api/accounts_mgmt/v1/subscriptions/{{ sub_id_result.stdout }} \
          --body='{{ payload | to_json }}'
      vars:
        payload:
          support_level: "{{ support_level }}"
          service_level: "{{ service_level }}"
          usage: "{{ usage }}"
          system_units: "{{ system_units }}"

    - name: Confirm settings applied
      ansible.builtin.debug:
        msg: "OCM subscription configured for {{ cluster_name }}: {{ support_level }} / {{ usage }}"
```

### Step 4 — Create the ClusterCurator Resource

Apply this to the RHACM hub alongside your cluster deployment:

```yaml
apiVersion: cluster.open-cluster-management.io/v1beta1
kind: ClusterCurator
metadata:
  name: prod-bm-01
  namespace: prod-bm-01
spec:
  desiredCuration: install
  install:
    prehook: []                          # No pre-install automation yet
    posthook:
      - name: set-ocm-subscription       # Job Template name in AAP
        extra_vars:
          cluster_name: prod-bm-01
          support_level: Premium
          service_level: L1-L3
          usage: Production
          system_units: Cores/vCPU
  towerAuthSecret: aap-credentials
```

### Step 5 — Monitor Execution

```bash
# Watch ClusterCurator status
oc get clustercurator prod-bm-01 -n prod-bm-01 -w

# View generated AnsibleJob
oc get ansiblejob -n prod-bm-01

# Describe for details
oc describe clustercurator prod-bm-01 -n prod-bm-01
```

## ClusterCurator Without AAP (Direct Script Invocation)

If you are not yet using Ansible Automation Platform, ClusterCurator can invoke hooks as Kubernetes Jobs instead. This is a lighter-weight approach suitable for getting started.

> **Note**: Direct job execution (without AAP) requires a custom operator setup and is less commonly documented. The AAP integration is the primary supported path. If AAP is not available, the standalone script run from a CI/CD pipeline (Tekton, Jenkins, GitHub Actions) is the practical alternative.

### Practical Alternative: Tekton Pipeline as Post-Install Hook

```yaml
# tekton/post-install-pipeline.yaml
apiVersion: tekton.dev/v1beta1
kind: Pipeline
metadata:
  name: post-cluster-install
  namespace: open-cluster-management
spec:
  params:
    - name: cluster-name
    - name: support-level
      default: Premium
    - name: usage
      default: Production
  tasks:
    - name: set-ocm-subscription
      taskRef:
        name: run-ocm-subscription-script
      params:
        - name: cluster-name
          value: $(params.cluster-name)
        - name: support-level
          value: $(params.support-level)
        - name: usage
          value: $(params.usage)
```

## ClusterCurator for Full Bare Metal Lifecycle

Once you are comfortable with post-install hooks, expand to the full lifecycle:

```yaml
apiVersion: cluster.open-cluster-management.io/v1beta1
kind: ClusterCurator
metadata:
  name: prod-bm-01
  namespace: prod-bm-01
spec:
  desiredCuration: install
  install:
    prehook:
      - name: validate-dns-entries         # Verify DNS before install
          extra_vars:
            api_vip: 10.20.30.10
            ingress_vip: 10.20.30.11
            cluster_name: prod-bm-01
      - name: create-cmdb-record           # Register in asset management
          extra_vars:
            cluster_name: prod-bm-01
            environment: production
    posthook:
      - name: set-ocm-subscription         # Set Red Hat subscription
          extra_vars:
            cluster_name: prod-bm-01
            support_level: Premium
      - name: configure-ldap-auth          # Set up LDAP authentication
          extra_vars:
            cluster_name: prod-bm-01
      - name: notify-teams-channel         # Alert team in Slack/Teams
          extra_vars:
            cluster_name: prod-bm-01
            message: "Cluster prod-bm-01 is ready"
  upgrade:
    prehook:
      - name: pre-upgrade-health-check
    posthook:
      - name: post-upgrade-validation
  destroy:
    prehook:
      - name: backup-etcd
      - name: notify-decommission
    posthook:
      - name: remove-cmdb-record
      - name: release-ip-reservations
  towerAuthSecret: aap-credentials
```

## Comparison: Current vs. ClusterCurator

| Aspect | Current (Manual / Script) | With ClusterCurator |
|---|---|---|
| Trigger | Manual after install | Automatic on cluster ready |
| Operator | Human runs script | RHACM controller triggers job |
| Scale | One at a time | All clusters automatically |
| Auditability | Shell history | AnsibleJob resources + AAP logs |
| Error handling | Manual retry | AAP retry logic |
| Extensibility | Shell script only | Any Ansible playbook |
| GitOps | Script in repo | ClusterCurator CRD in Git |

## Getting Started with ClusterCurator — Recommended Path

### Phase 1 (Now): Standalone Script
Run `set-ocm-subscription.sh` manually or from your CI/CD pipeline after each cluster install. Low setup cost, immediate benefit.

### Phase 2 (Next): ClusterCurator + AAP
Once you have AAP deployed, convert the standalone script into an Ansible playbook and create `ClusterCurator` resources alongside each cluster deployment.

### Phase 3 (Future): Full Lifecycle Automation
Expand prehooks and posthooks to cover DNS validation, CMDB registration, LDAP configuration, notifications, and decommission workflows.

## Checking ClusterCurator Is Available

```bash
# Check if cluster-curator-controller is installed
oc get deployment cluster-curator-controller -n open-cluster-management

# Check CRD is present
oc get crd clustercurators.cluster.open-cluster-management.io

# View existing ClusterCurators
oc get clustercurator --all-namespaces
```

If the controller is not present, it ships with RHACM 2.2+ and should be installed by default. Check your MCE/RHACM operator subscription.
